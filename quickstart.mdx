## 開発環境のセットアップ

ドキュメントをローカルで更新し、公開する方法を学びましょう。

### 編集とプレビュー

<AccordionGroup>
  <Accordion icon="github" title="ドキュメントをローカルにクローンする">
    オンボーディングプロセス中に、GitHubにドキュメントのコンテンツを含むリポジトリを作成しました。
    このリポジトリは[ダッシュボード](https://dashboard.mintlify.com)で確認できます。
    リポジトリをローカルにクローンするには、ターミナルでこちらの
    [手順](https://docs.github.com/en/repositories/creating-and-managing-repositories/cloning-a-repository)
    に従ってください。
  </Accordion>
  <Accordion icon="rectangle-terminal" title="変更をプレビューする">
    プレビューは、変更が意図した通りに表示されることを確認するのに役立ちます。
    これらの変更をローカルでレンダリングするためのコマンドラインインターフェースを構築しました。
    1. [Mintlify CLI](https://www.npmjs.com/package/mintlify)をインストールして、
    以下のコマンドでドキュメントの変更をローカルでプレビューできます：
    ``` npm i -g mintlify ```
    2. ドキュメントのルートディレクトリ（`mint.json`がある場所）で
    以下のコマンドを実行してください：
    ``` mintlify dev ```
  </Accordion>
</AccordionGroup>

### 変更のデプロイ

<AccordionGroup>

<Accordion icon="message-bot" title="GitHubアプリをインストール">
  当社のGitHubアプリは、ドキュメントサイトへの変更を自動的にデプロイするため、
  デプロイメントを自分で管理する必要はありません。
  インストールリンクは[ダッシュボード](https://dashboard.mintlify.com)で確認できます。
  ボットが正常にインストールされると、リポジトリのコミットハッシュの横にチェックマークが表示されます。
</Accordion>
<Accordion icon="rocket" title="変更をプッシュ">
  ドキュメントサイトを更新するには、[Gitに変更をコミットしてプッシュ](https://docs.github.com/en/get-started/using-git/pushing-commits-to-a-remote-repository#about-git-push)
  してください。プッシュ後にGitHubアプリが変更を正常にデプロイしていない場合は、
  [ダッシュボード](https://dashboard.mintlify.com)から手動でドキュメントを更新することもできます。
</Accordion>

</AccordionGroup>
