<Info>
  **前提条件**: 続行する前にNode.js（バージョン19以上）をインストールしてください。
</Info>

お使いのオペレーティングシステムにMintlifyをインストールして実行するには、以下の手順に従ってください：

**ステップ1**: Mintlifyのインストール：

<CodeGroup>

  ```bash npm
  npm i -g mintlify
  ```

```bash yarn
yarn global add mintlify
```

</CodeGroup>

**ステップ2**: docsディレクトリ（`mint.json`ファイルがある場所）に移動し、次のコマンドを実行します：

```bash
mintlify dev
```

ドキュメントのローカルプレビューは`http://localhost:3000`で利用可能になります。

### カスタムポート

デフォルトでは、Mintlifyはポート3000を使用します。`--port`フラグを使用してMintlifyが実行されるポートをカスタマイズできます。例えば、ポート3333でMintlifyを実行するには、次のコマンドを使用します：

```bash
mintlify dev --port 3333
```

すでに使用中のポートでMintlifyを実行しようとすると、次の利用可能なポートを使用します：

```md
ポート3000はすでに使用されています。代わりに3001を試します。
```

## Mintlifyのバージョン

各CLIリリースは特定のMintlifyバージョンに関連付けられていることにご注意ください。ローカルウェブサイトが本番バージョンと一致しない場合は、CLIを更新してください：

<CodeGroup>

```bash npm
npm i -g mintlify@latest
```

```bash yarn
yarn global upgrade mintlify
```

</CodeGroup>

## リンクの検証

CLIはドキュメント内で作成された参照リンクの検証を支援します。破損したリンクを特定するには、次のコマンドを使用します：

```bash
mintlify broken-links
```

## コードフォーマット

MDXを認識してフォーマットするためのIDE拡張機能の使用をお勧めします。VSCodeユーザーの場合、構文ハイライト用の[MDX VSCode拡張機能](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx)と、コードフォーマット用の[Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode)をご検討ください。

## トラブルシューティング

<AccordionGroup>
  <Accordion title='エラー: darwin-arm64ランタイムを使用して"sharp"モジュールを読み込めませんでした'>

    これは古いバージョンのnodeが原因である可能性があります。以下を試してください：
    1. 現在インストールされているmintlifyバージョンを削除：`npm remove -g mintlify`
    2. Node v19以上にアップグレード。
    3. mintlifyを再インストール：`npm install -g mintlify`
  </Accordion>

  <Accordion title="問題: 不明なエラーが発生した場合">
  
    解決策：デバイスのルートに移動し、~/.mintlifyフォルダを削除します。その後、`mintlify dev`を再度実行してください。
  </Accordion>
</AccordionGroup>
