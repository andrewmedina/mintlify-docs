<Info>
  **Prasyarat**: Mangga install Node.js (versi 19 utawa luwih dhuwur) sadurunge nerusake.
</Info>

Tutna langkah-langkah iki kanggo masang lan mbukak Mintlify ing sistem operasi sampeyan:

**Langkah 1**: Masang Mintlify:

<CodeGroup>

  ```bash npm
  npm i -g mintlify
  ```

```bash yarn
yarn global add mintlify
```

</CodeGroup>

**Langkah 2**: Pindhah menyang direktori docs (ing ngendi file `mint.json` manggon) lan jalanke printah iki:

```bash
mintlify dev
```

Preview lokal saka dokumentasi sampeyan bakal kasedhiya ing `http://localhost:3000`.

### Port Kustom

Standar, Mintlify nggunakake port 3000. Sampeyan bisa ngowahi port sing digunakake Mintlify nganggo tandha `--port`. Kanggo mbukak Mintlify ing port 3333, contone, gunakake printah iki:

```bash
mintlify dev --port 3333
```

Yen sampeyan nyoba mbukak Mintlify ing port sing wis digunakake, bakal nggunakake port sing kasedhiya sabanjure:

```md
Port 3000 wis digunakake. Nyoba 3001.
```

## Versi Mintlify

Mangga cathet menawa saben release CLI ana hubungane karo versi tartamtu saka Mintlify. Yen website lokal sampeyan ora cocog karo versi produksi, mangga nganyari CLI:

<CodeGroup>

```bash npm
npm i -g mintlify@latest
```

```bash yarn
yarn global upgrade mintlify
```

</CodeGroup>

## Validasi Link

CLI bisa mbantu validasi referensi link sing digawe ing dokumentasi sampeyan. Kanggo ngenali link sing rusak, gunakake printah iki:

```bash
mintlify broken-links
```

## Format Kode

Kita nyaranake nggunakake ekstensi ing IDE sampeyan kanggo ngenali lan format MDX. Yen sampeyan pengguna VSCode, pertimbangake [MDX VSCode extension](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) kanggo sorotan sintaks, lan [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) kanggo format kode.

## Ngatasi Masalah

<AccordionGroup>
  <Accordion title='Kesalahan: Ora bisa ngemot modul "sharp" nggunakake runtime darwin-arm64'>

    Iki bisa amarga versi node sing lawas. Coba langkah-langkah iki:
    1. Mbusak versi mintlify sing wis dipasang: `npm remove -g mintlify`
    2. Upgrade menyang Node v19 utawa luwih dhuwur.
    3. Masang maneh mintlify: `npm install -g mintlify`
  </Accordion>

  <Accordion title="Masalah: Nemokake kesalahan sing ora dingerteni">
  
    Solusi: Pindhah menyang root piranti sampeyan lan busak folder ~/.mintlify. Sabanjure, jalanke `mintlify dev` maneh.
  </Accordion>
</AccordionGroup>

Penasaran karo apa sing owah ing versi CLI? [Deleng changelog CLI.](https://www.npmjs.com/package/mintlify?activeTab=versions)
