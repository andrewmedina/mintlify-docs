description: 'Mintlify geeft je volledige controle over het uiterlijk van je documentatie via het mint.json bestand'
icon: 'gear'
---

Elke Mintlify site heeft een `mint.json` bestand nodig met de belangrijkste configuratie-instellingen. Leer meer over de [eigenschappen](#properties) hieronder.

## Eigenschappen

<ResponseField name="name" type="string" required>
Naam van je project. Wordt gebruikt voor de globale titel.

Voorbeeld: `mintlify`

</ResponseField>

<ResponseField name="navigation" type="Navigation[]" required>
  Een array van groepen met alle pagina's binnen die groep
  <Expandable title="Navigation">
    <ResponseField name="group" type="string">
    De naam van de groep.

    Voorbeeld: `Instellingen`

    </ResponseField>
    <ResponseField name="pages" type="string[]">
    De relatieve paden naar de markdown bestanden die als pagina's zullen dienen.

    Voorbeeld: `["customization", "page"]`

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="logo" type="string or object">
  Pad naar het logo of object met pad naar "light" en "dark" mode logo afbeeldingen
  <Expandable title="Logo">
    <ResponseField name="light" type="string">
      Pad naar het logo in light mode
    </ResponseField>
    <ResponseField name="dark" type="string">
      Pad naar het logo in dark mode
    </ResponseField>
    <ResponseField name="href" type="string" default="/">
      Waar het klikken op het logo naar toe leidt
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string">
  Pad naar de favicon afbeelding
</ResponseField>

<ResponseField name="colors" type="Colors">
  Hex-kleurcodes voor je globale thema
  <Expandable title="Colors">
    <ResponseField name="primary" type="string" required>
      De primaire kleur. Meestal gebruikt voor gemarkeerde inhoud, sectiekoppen en accenten in light mode
    </ResponseField>
    <ResponseField name="light" type="string">
      De primaire kleur voor dark mode. Meestal gebruikt voor gemarkeerde inhoud, sectiekoppen en accenten in dark mode
    </ResponseField>
    <ResponseField name="dark" type="string">
      De primaire kleur voor belangrijke knoppen
    </ResponseField>
    <ResponseField name="background" type="object">
      De achtergrondkleur in zowel light als dark mode
      <Expandable title="Object">
        <ResponseField name="light" type="string" required>
          De hex-kleurcode van de achtergrond in light mode
        </ResponseField>
        <ResponseField name="dark" type="string" required>
          De hex-kleurcode van de achtergrond in dark mode
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarLinks" type="TopbarLink[]">
  Array van `name`s en `url`s van links die je wilt opnemen in de topbalk
  <Expandable title="TopbarLink">
    <ResponseField name="name" type="string">
    De naam van de knop.

    Voorbeeld: `Contact`
    </ResponseField>
    <ResponseField name="url" type="string">
    De url wanneer je op de knop klikt. Voorbeeld: `https://mintlify.com/docs`
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="topbarCtaButton" type="Call to Action">
  <Expandable title="Topbar Call to Action">
    <ResponseField name="type" type={'"link" or "github"'} default="link">
    Link toont een knop. GitHub toont de repo-informatie op de opgegeven url, inclusief het aantal GitHub-sterren.
    </ResponseField>
    <ResponseField name="url" type="string">
    Bij `link`: Waar de knop naar verwijst.
    
    Bij `github`: Link naar de repository om GitHub-informatie van te laden.
    </ResponseField>
    <ResponseField name="name" type="string">
    Tekst in de knop. Alleen vereist als `type` een `link` is.
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="versions" type="string[]">
  Array van versienamen. Gebruik dit alleen als je verschillende versies van documentatie wilt tonen met een dropdown in de navigatiebalk.
</ResponseField>

<ResponseField name="anchors" type="Anchor[]">
  Een array van ankers, bevat de `icon`, `color`, en `url`.
  <Expandable title="Anchor">
    <ResponseField name="icon" type="string">
    Het [Font Awesome](https://fontawesome.com/search?q=heart) icoon gebruikt voor het anker.

    Voorbeeld: `comments`
    </ResponseField>
    <ResponseField name="name" type="string">
    De naam van het ankerlabel.

    Voorbeeld: `Community`
    </ResponseField>
    <ResponseField name="url" type="string">
      Het begin van de URL dat aangeeft welke pagina's bij het anker horen. Dit is meestal de naam van de map waarin je je pagina's plaatst.
    </ResponseField>
    <ResponseField name="color" type="string">
      De hex-kleur van de anker-icoon achtergrond. Kan ook een verloop zijn als je een object doorgeeft met de eigenschappen `from` en `to` die elk een hex-kleur zijn.
    </ResponseField>
    <ResponseField name="version" type="string">
      Gebruikt als je een anker wilt verbergen tot de juiste docs-versie is geselecteerd.
    </ResponseField>
    <ResponseField name="isDefaultHidden" type="boolean" default="false">
      Stel in op `true` als je het anker wilt verbergen totdat je iemand direct linkt naar docs erin.
    </ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">
      Een van: "brands", "duotone", "light", "sharp-solid", "solid", of "thin"
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="topAnchor" type="Object">
  Overschrijf de standaardconfiguraties voor het bovenste anker.
  <Expandable title="Object">
    <ResponseField name="name" default="Documentation" type="string">
      De naam van het bovenste anker
    </ResponseField>
    <ResponseField name="icon" default="book-open" type="string">
      Font Awesome icoon.
    </ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">
      Een van: "brands", "duotone", "light", "sharp-solid", "solid", of "thin"
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="tabs" type="Tabs[]">
  Een array van navigatietabs.
  <Expandable title="Tabs">
    <ResponseField name="name" type="string">
      De naam van het tablabel.
    </ResponseField>
    <ResponseField name="url" type="string">
      Het begin van de URL dat aangeeft welke pagina's bij de tab horen. Dit is meestal de naam van de map waarin je je pagina's plaatst.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="api" type="API">
  Configuratie voor API-instellingen. Leer meer over API-pagina's bij [API Components](/api-playground/demo).
  <Expandable title="API">
    <ResponseField name="baseUrl" type="string">
      De basis-url voor alle API-endpoints. Als `baseUrl` een array is, maakt het meerdere basis-url opties mogelijk die de gebruiker kan wisselen.
    </ResponseField>

    <ResponseField name="auth" type="Auth">
      <Expandable title="Auth">
        <ResponseField name="method" type='"bearer" | "basic" | "key"'>
          De authenticatiestrategie gebruikt voor alle API-endpoints.
        </ResponseField>
        <ResponseField name="name" type="string">
        De naam van de authenticatieparameter gebruikt in de API-playground.

        Als de methode `basic` is, moet het formaat `[gebruikersnaamNaam]:[wachtwoordNaam]` zijn
        </ResponseField>
        <ResponseField name="inputPrefix" type="string">
        De standaardwaarde die bedoeld is als voorvoegsel voor het authenticatie-invoerveld.

        Bijv. Een `inputPrefix` van `AuthKey` zou het standaard invoerresultaat van het authenticatieveld erven als `AuthKey`.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="playground" type="Playground">
      Configuraties voor de API-playground

      <Expandable title="Playground">
        <ResponseField name="mode" default="show" type='"show" | "simple" | "hide"'>
          Of de playground wordt getoond, verborgen is, of alleen het endpoint toont zonder toegevoegde gebruikersinteractiviteit `simple`

          Leer meer in de [playground handleidingen](/api-playground/demo)
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="maintainOrder" type="boolean">
      Het inschakelen van deze vlag zorgt ervoor dat de sleutelvolgorde in OpenAPI-pagina's overeenkomt met de sleutelvolgorde gedefinieerd in het OpenAPI-bestand.

      <Warning>Dit gedrag zal binnenkort standaard worden ingeschakeld, waarna dit veld zal worden afgekeurd.</Warning>
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="openapi" type="string | string[]">
  Een string of een array van strings van URL(s) of relatieve pad(en) die naar je OpenAPI-bestand wijzen.
  
  Voorbeelden:
  <CodeGroup>
    ```json Absolute
    "openapi": "https://example.com/openapi.json"
    ```
    ```json Relative
    "openapi": "/openapi.json"
    ```
    ```json Multiple
    "openapi": ["https://example.com/openapi1.json", "/openapi2.json", "/openapi3.json"]
    ```
  </CodeGroup>

</ResponseField>

<ResponseField name="footerSocials" type="FooterSocials">
  Een object van sociale media-accounts waarbij het sleutel:eigenschap-paar het sociale mediaplatform en de account-url vertegenwoordigt.
  
  Voorbeeld: 
  ```json
  {
    "x": "https://x.com/mintlify",
    "website": "https://mintlify.com"
  }
  ```
  <Expandable title="FooterSocials">
    <ResponseField name="[key]" type="string">
    Een van de volgende waarden `website`, `facebook`, `x`, `discord`, `slack`, `github`, `linkedin`, `instagram`, `hacker-news`
    
    Voorbeeld: `x`
    </ResponseField>
    <ResponseField name="property" type="string">
    De URL naar het sociale platform.
    
    Voorbeeld: `https://x.com/mintlify`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="feedback" type="Feedback">
  Configuraties om feedback-knoppen in te schakelen

  <Expandable title="Feedback">
    <ResponseField name="suggestEdit" type="boolean" default="false">
    Schakelt een knop in waarmee gebruikers bewerkingen kunnen voorstellen via pull requests
    </ResponseField>
    <ResponseField name="raiseIssue" type="boolean" default="false">
    Schakelt een knop in waarmee gebruikers een probleem kunnen melden over de documentatie
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="modeToggle" type="ModeToggle">
  Pas de dark mode schakelaar aan.
  <Expandable title="ModeToggle">
    <ResponseField name="default" type={'"light" or "dark"'}>
      Stel in als je altijd light of dark mode wilt tonen voor nieuwe gebruikers. Wanneer niet ingesteld, gebruiken we standaard dezelfde modus als het besturingssysteem van de gebruiker.
    </ResponseField>
    <ResponseField name="isHidden" type="boolean" default="false">
      Stel in op true om de dark/light mode schakelaar te verbergen. Je kunt `isHidden` combineren met `default` om je docs te dwingen alleen light of dark mode te gebruiken. Bijvoorbeeld:
      
      <CodeGroup>
      ```json Alleen Dark Mode
      "modeToggle": {
        "default": "dark",
        "isHidden": true
      }
      ```

      ```json Alleen Light Mode
      "modeToggle": {
        "default": "light",
        "isHidden": true
      }
      ```
      </CodeGroup>

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
  Een achtergrondafbeelding die achter elke pagina wordt weergegeven. Zie voorbeeld bij
  [Infisical](https://infisical.com/docs) en [FRPC](https://frpc.io).
</ResponseField>
