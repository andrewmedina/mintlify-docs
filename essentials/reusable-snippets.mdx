## カスタムスニペットの作成

**前提条件**: スニペットファイルは`snippets`ディレクトリに作成する必要があります。

<Note>
  `snippets`ディレクトリ内のページはすべてスニペットとして扱われ、単独のページとして
  レンダリングされません。スニペットから単独のページを作成したい場合は、スニペットを
  別のファイルにインポートし、コンポーネントとして呼び出してください。
</Note>

### デフォルトエクスポート

1. 複数の場所で再利用したいコンテンツをスニペットファイルに追加します。
   必要に応じて、スニペットをインポートする際にpropsを通じて入力できる
   変数を追加することもできます。

```mdx snippets/my-snippet.mdx
こんにちは！これは複数のページで再利用したいコンテンツです。今日のキーワードは{word}です。
```

<Warning>
  再利用したいコンテンツは、インポートが機能するために`snippets`ディレクトリ内に
  配置する必要があります。
</Warning>

2. スニペットを目的のファイルにインポートします。

```mdx destination-file.mdx
---
title: タイトル
description: 説明文
---

import MySnippet from '/snippets/path/to/my-snippet.mdx';

## ヘッダー

Lorem impsum dolor sit amet.

<MySnippet word="バナナ" />
```

### 再利用可能な変数

1. スニペットファイルから変数をエクスポートします：

```mdx snippets/path/to/custom-variables.mdx
export const myName = '私の名前';

export const myObject = { fruit: 'いちご' };
```

2. 目的のファイルからスニペットをインポートし、変数を使用します：

```mdx destination-file.mdx
---
title: タイトル
description: 説明文
---

import { myName, myObject } from '/snippets/path/to/custom-variables.mdx';

こんにちは、私の名前は{myName}で、{myObject.fruit}が好きです。
```

### 再利用可能なコンポーネント

1. スニペットファイル内で、アロー関数の形式でコンポーネントをエクスポートすることで、
   propsを受け取るコンポーネントを作成します。

```mdx snippets/custom-component.mdx
export const MyComponent = ({ title }) => (
  <div>
    <h1>{title}</h1>
    <p>... スニペットの内容 ...</p>
  </div>
);
```

<Warning>
  MDXはアロー関数の本体内ではコンパイルされません。可能な限りHTML構文を使用するか、
  MDXを使用する必要がある場合はデフォルトエクスポートを使用してください。
</Warning>

2. スニペットを目的のファイルにインポートし、propsを渡します

```mdx destination-file.mdx
---
title: タイトル
description: 説明文
---

import { MyComponent } from '/snippets/custom-component.mdx';

Lorem ipsum dolor sit amet.

<MyComponent title={'カスタムタイトル'} />
```
